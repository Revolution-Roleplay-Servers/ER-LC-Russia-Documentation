# 🚨 Лимиты Запросов

{% hint style="warning" %}
**ER:LC Россия API не поддерживает хостинги ботов, такие как Bot Ghost.**

Сервисы вроде BotGhost хостят много приложений под одним IP, что приводит к частым Rate Limit. Не используйте BotGhost с нашим API.
{% endhint %}

<h3 align="center">Глобальный Rate Limit</h3>

| Категория эндпоинтов                | Лимит                                    |
| ----------------------------------- | ---------------------------------------- |
| Stats API V1 (`/v1/moscowroleplay/stats/*`)           | **20 запросов/мин**                      |
| Stats API V2 (`/v2/moscowroleplay/stats/*`)        | **40 запросов/мин** (без токена)         |
| Stats API V2 (`/v2/moscowroleplay/stats/*`)        | **120 запросов/мин** (с токеном)         |
| ER:LC API (`/v1/moscowroleplay/erlc/*`, `/v2/moscowroleplay/erlc/*`) | **120 запросов/мин**                     |

<h4 align="center">Лимиты По Эндпоинтам</h4>

| Эндпоинт                   | Лимит      | Описание                       |
| -------------------------- | ---------- | ------------------------------ |
| `/v1/moscowroleplay/erlc/*` | 120/мин    | Все эндпоинты ER:LC API V1     |
| `/v2/moscowroleplay/erlc/*` | 120/мин    | Все эндпоинты ER:LC API V2     |
| `/v1/moscowroleplay/stats/*` | 20/мин     | Stats V1 (отдельные эндпоинты) |
| `/v2/moscowroleplay/stats/*` | 40/120/мин | Stats V2 (без токена / с токеном) |

<h3 align="center">Защита От Спама</h3>

После превышения лимита (20, 40 или 120 запросов/мин) счётчик **не останавливается**, а продолжает считать **лишние запросы**.

При достижении **400 лишних запросов** (сверх лимита) выдаётся **бан на 5 минут**. Если спам продолжается, время бана увеличивается.

> **Stats V2:** Используйте токен для увеличения лимита с 40 до 120 запросов/мин. Передайте токен через заголовок `Authorization: Bearer <token>`.

<h4 align="center"><strong>Механика Анти-Cпама</strong></h4>

| Порог лишних запросов | Наказание                          |
| --------------------- | ---------------------------------- |
| **400+**              | Бан на **5 минут**                 |
| **500+**              | Бан на **6 минут** (+1 минута)     |
| **600+**              | Бан на **7 минут** (+2 минуты)     |
| **700+**              | Бан на **8 минут** (+3 минуты)     |
| ...                   | +1 минута за каждые 100 превышения |

> **Важно:** Если во время бана вы продолжаете отправлять запросы, время блокировки **продлевается** на момент обработки новых лишних запросов.

<h3 align="center"><strong>Ответ При Превышении Лимита</strong></h3>

```json
{
  "code": 5016,
  "message": "rate_limit_exceeded",
  "retry_after": 45,
  "details": {
    "limit": 120,
    "count": 125,
    "excess": 5,
    "reset_at": 1740339600000
  }
}
```

<h4 align="center"><strong>Ответ При Блокировке За Спам</strong></h4>

```json
{
  "code": 5023,
  "message": "spam_blocked",
  "blocked_until": 1740339600000,
  "remaining_seconds": 300
}
```

<h4 align="center"><strong>Ответ При Активной Блокировке</strong></h4>

```json
{
  "code": 5023,
  "message": "spam_blocked",
  "blocked_until": 1740339600000,
  "remaining_seconds": 245
}
```

> **Примечание:** `blocked_until` возвращается в формате **Unix timestamp (мс)**, а не ISO-дата.

<h4 align="center"><strong>Заголовки Rate Limit</strong></h4>

API возвращает заголовки с информацией о лимитах в **каждом ответе**:

| Заголовок               | Пример значения  | Описание                                        |
| ----------------------- | ---------------- | ----------------------------------------------- |
| `X-RateLimit-Limit`     | 120              | Максимальное количество запросов в минуту       |
| `X-RateLimit-Remaining` | 115              | Оставшееся количество запросов до сброса        |
| `X-RateLimit-Reset`     | _Unix timestamp_ | Время (мс), когда лимит будет сброшен           |
| `Retry-After`           | 45               | Секунд до сброса лимита (только при 429 ошибке) |

<h4 align="center">Очередь Команд</h4>

Эндпоинт `/erlc/server/command` использует систему очереди:

* **Максимальный размер очереди:** 100 команд
* **Интервал выполнения:** 1 команда каждые 1.5 секунды

Когда очередь заполнена, вы получите:

```json
{
  "code": 5015,
  "message": "CommandQueueFull"
}
```
