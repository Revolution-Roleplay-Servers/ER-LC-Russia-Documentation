# 🚧 Коды Ошибок

<h3 align="center">Формат Ответа Об Ошибке</h3>

Большинство ошибок возвращаются в формате JSON с полем `detail` и дополнительными полями.

<h4 align="center">Стандартные Ошибки API</h4>

```json
{
    "detail": "описание_ошибки"
}
```

<h4 align="center">Ошибки PRC API (ER:LC)</h4>

Ошибки от внешнего API Police RolePlay Community возвращаются с дополнительными полями:

```json
{
    "detail": "prc_api_error",
    "code": 2002,
    "message": "Invalid or expired server-key"
}
```

<h4 align="center">Ошибки Rate Limit</h4>

```json
{
    "detail": "rate_limit_exceeded",
    "bucket": "global",
    "retry_after": 30
}
```

<h3 align="center">Таблица Всех Кодов Ошибок</h3>

<h4 align="center">Ошибки PRC API (External ER:LC API)</h4>

Эти коды ошибок возвращаются от внешнего API Police RolePlay Community при запросах к эндпоинтам `/erlc/server/*`.

| Код      | Название                   | Описание                                                                              | Повторять? | Решение                                                |
| -------- | -------------------------- | ------------------------------------------------------------------------------------- | ---------- | ------------------------------------------------------ |
| **0**    | Unknown Error              | Неизвестная ошибка. Если вы столкнулись с этой ошибкой, откройте тикет в PRC Discord. | ❌ Нет      | Проверьте логи, обратитесь в поддержку PRC             |
| **1001** | Roblox Communication Error | Ошибка связи с Roblox или приватным сервером                                          | ✅ Да       | Подождите несколько секунд, попробуйте снова           |
| **1002** | Internal System Error      | Внутренняя системная ошибка PRC API                                                   | ✅ Да       | Повторите запрос через несколько секунд                |
| **2000** | No Server-Key              | Не предоставлен Server-Key в заголовке                                                | ❌ Нет      | Добавьте заголовок `Server-Key: <ваш_ключ>`            |
| **2001** | Invalid Server-Key Format  | Неверный формат Server-Key                                                            | ❌ Нет      | Проверьте формат ключа (должен быть валидным)          |
| **2002** | Invalid/Expired Server-Key | Недействительный или истёкший Server-Key                                              | ❌ Нет      | **Сгенерируйте новый Server-Key** в настройках сервера |
| **2003** | Invalid Global API Key     | Недействительный глобальный API ключ                                                  | ❌ Нет      | Проверьте ваш API ключ                                 |
| **2004** | Server-Key Banned          | Server-Key заблокирован для доступа к API                                             | ❌ Нет      | Обратитесь в поддержку PRC                             |
| **3001** | Invalid Command            | Неверная команда в запросе                                                            | ❌ Нет      | Проверьте синтаксис команды                            |
| **3002** | Server Offline             | Сервер отключён (нет игроков)                                                         | ✅ Да       | Дождитесь запуска сервера                              |
| **4001** | Rate Limited               | Превышен лимит запросов                                                               | ✅ Да       | Дождитесь сброса лимита (см. `retry_after`)            |
| **4002** | Command Restricted         | Команда заблокирована/ограничена                                                      | ❌ Нет      | Используйте другую команду                             |
| **4003** | Message Prohibited         | Сообщение запрещено                                                                   | ❌ Нет      | Измените текст сообщения                               |
| **9998** | Resource Restricted        | Ресурс ограничен                                                                      | ❌ Нет      | Проверьте права доступа                                |
| **9999** | Module Out of Date         | Модуль на сервере устарел                                                             | ✅ Да       | Обновите модуль на сервере                             |

> **Повторяемые ошибки** — при получении таких ошибок API автоматически попытается повторить запрос (до 3 раз). Для неповторяемых ошибок повтор запроса не имеет смысла.

<h4 align="center">Общие Ошибки API</h4>

| Код     | Статус                | Описание              | Причина и решение                                                                 |
| ------- | --------------------- | --------------------- | --------------------------------------------------------------------------------- |
| **429** | Too Many Requests     | `rate_limit_exceeded` | Превышен лимит запросов. Проверьте заголовки `X-RateLimit-*` и поле `retry_after` |
| **500** | Internal Server Error | `internal_error`      | Непредвиденная ошибка на стороне сервера. Ошибка логируется                       |
| **0**   | Unknown Error         | (Нестандартный код)   | Неизвестная ошибка. Откройте тикет с подробностями                                |

<p align="center"><strong>Что делать при неизвестной ошибке (код 0)?</strong></p>

Пожалуйста, **откройте тикет** и предоставьте:

* Какой эндпоинт вызывали
* Какие параметры передавали (кроме секретных данных)
* Время возникновения ошибки
* Полученный ответ от API

<h4 align="center">Ошибки Аутентификации и Доступа</h4>

| Эндпоинт         | Код | Статус       | Описание        | Причина и решение                                             |
| ---------------- | --- | ------------ | --------------- | ------------------------------------------------------------- |
| `/erlc/server/*` | 401 | Unauthorized | `missing_token` | Отсутствует API ключ. Добавьте `Authorization: Bearer <ключ>` |
| `/erlc/server/*` | 401 | Unauthorized | `invalid_token` | API ключ недействителен или деактивирован                     |
| `/erlc/server/*` | 403 | Forbidden    | `forbidden`     | Нет доступа к этому эндпоинту. Обратитесь к администратору    |

<h3 align="center">Ошибки Клиента</h3>

| Эндпоинт               | Код | Статус            | Описание                                     | Причина и решение                                                                |
| ---------------------- | --- | ----------------- | -------------------------------------------- | -------------------------------------------------------------------------------- |
| `/stats/*`             | 400 | Bad Request       | `Неподдерживаемый сервер`                    | Неверное имя сервера. Доступные: `MVD`, `FSB`, `MCHS`, `MZ`, `112`, `DOT`, `MRP` |

| `/erlc/server/command` | 429 | Too Many Requests | `CommandQueueFull`                           | Очередь команд заполнена (макс. 100 команд)                                      |

<h3 align="center">Рекомендации По Обработке Ошибок</h3>

#### Для разработчиков клиентов

1. **Всегда проверяйте код ошибки** — не только HTTP статус, но и поле `code` в теле ответа
2. **Уважайте `retry_after`** — при получении 429 дождитесь указанного времени
3. **Не повторяйте неповторяемые ошибки** — коды 2000-2004, 3001, 4002-4003, 9998 не требуют повторной попытки
4. **Логгируйте ошибки** — сохраняйте полную информацию об ошибках для отладки
5. **Используйте кеш** — при временной недоступности API можно использовать закешированные данные

<h4 align="center">Автоматические Повторные Попытки</h4>

API автоматически повторяет запросы только для **повторяемых** ошибок:

* 1001, 1002 (ошибки связи)
* 3002 (сервер офлайн)
* 4001 (Rate Limit)
* 9999 (модуль устарел)

Максимальное количество попыток: **3** с экспоненциальной задержкой.
